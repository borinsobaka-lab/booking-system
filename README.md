# Запись на массаж — онлайн-запись + админка

Система онлайн-записи на услуги (массаж): витрина для клиентов и админка для
сотрудников. Фронтенд на React + Vite + TypeScript. Данные пока хранятся локально
в браузере (`localStorage`) — это рабочая «симуляция» без сервера: приложение
полностью функционирует и демонстрируется. Слой данных изолирован
(`src/db.ts`), поэтому позже его можно заменить на настоящий бэкенд, не трогая UI.

## Что уже есть

### Админка (`#/admin`)
- **Первичная настройка**: при первом запуске создаётся **суперадминистратор**
  (owner) — вы задаёте его логин и пароль сами.
- **Пользователи** (только для суперадминистратора): заведение **админов** и
  **мастеров**, выдача им логинов и паролей (пароль показывается один раз, можно
  сгенерировать и скопировать), смена паролей, удаление. Админы и мастера входят
  и видят записи, но **не** управляют пользователями.
- **Услуги**: название, описание, длительность, стоимость, картинка.
- **Специалисты**: аватар, имя, фамилия, роль и список выполняемых услуг.
- **Расписание**: у каждого специалиста на недельном таймлайне мышью
  «протягиваются» рабочие часы; отдельным режимом добавляются перерывы. Видно,
  какие дни рабочие, а какие выходные; выходной легко сделать рабочим.
- **Записи**: большой календарь на день по каждому специалисту — рабочие окна,
  перерывы и записи клиентов. Здесь же можно создать запись вручную или отменить
  существующую.

### Витрина клиента (`#/`)
Баннер бренда (аватар, название, адрес) и три пути записи:
1. **Мастер** → услуга → дата и время.
2. **Выбрать дату** → время (только когда есть свободный специалист) → услуга →
   мастер (недоступные в это время — заблюрены).
3. **Выбрать услугу** → мастер → дата и время.

Доступны только реально свободные даты/времена; после выбора запись
**подтверждается автоматически** (симуляция). Email-уведомления и полноценная
регистрация брони — следующий этап.

## Разработка

```bash
npm install
npm run dev        # http://localhost:5173/task-tracker/
npm run build      # проверка типов + сборка в dist/
npm run test:e2e   # e2e-тесты Playwright
```

Приложение хэш-роутится: `#/` — витрина клиента, `#/admin` — админка.
